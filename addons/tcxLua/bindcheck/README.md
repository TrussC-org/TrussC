# bindcheck — tcxLua binding regression harness

Test app that verifies the tcxLua Lua bindings are **reachable and callable**.
Run it after regenerating or editing the bindings. It is the runtime companion to
the compile-witness: compiling proves the bindings are valid C++; bindcheck proves
they actually arrive in the Lua state and work when called.

## What it checks
1. **Existence** — every `set_function("name", …)` and every `new_usertype<…>("Name", …)`
   from all three binding sources is reachable as a Lua global (`_G[name] ~= nil`):
   - `src/generated/trussc_generated.cpp` (free functions, generated by `tools/luagen`)
   - `src/generated/trussctype_generated.cpp` (usertypes, generated by `tools/luagen-types`)
   - `src/tcxLua.cpp` (hand-written usertypes + custom glue)
2. **Callability** — a subset of pure functions and constructors is actually invoked
   and the results asserted. This includes regression checks for past codegen bugs:
   - `Color.new(1,0,0)` — 3-arg RGB ctor (trailing-default arity expansion)
   - `-Vec3.new(1,2,3)` — unary minus (operator with unary + binary overloads)
   - `Node.new()` / `Serial` — types once mis-excluded from generation

## Workflow after editing / regenerating the bindings
```bash
cd addons/tcxLua/bindcheck

# 1) Surface check — --regen rebuilds bin/data/expected.lua from the CURRENT binding
#    sources, then builds & runs this app and prints the report.
./run_bindcheck.sh --regen

# 2) Full example sweep — build & briefly run every tcxLua example, catching
#    crashes / sol::error at draw time that the static check can't see.
./sweep_examples.sh
```

### Interpreting `run_bindcheck.sh`
```
functions_present == functions_expected   # all free functions reachable
usertypes_present == usertypes_expected   # all usertypes reachable
call_failures == 0                        # call paths work
```
Any `MISSING_FN <name>` / `MISSING_UT <name>` / `CALL_FAIL <label>` line = a real
regression.

### Interpreting `sweep_examples.sh`
`BUILD=OK` and `RUNTIME=ALIVE` for every row = healthy. `CRASH(n)` (e.g. 134/SIGABRT)
= an uncaught `sol::error` at runtime → a broken binding call. Logs land in
`sweep_logs/`. Lesson learned: the existence check alone once stayed green while
`Color.new(1,0,0)` aborted at runtime — run the sweep (or at least one example)
for changes that touch constructors/overloads.

## Key files
- `src/tcApp.cpp` — loads `data/bindcheck.lua` in `setup()`, then self-exits
- `bin/data/bindcheck.lua` — verify logic (existence walk + `try(...)` call asserts)
- `bin/data/expected.lua` — expected names, auto-generated by `regen_expected.sh`
  (git-ignored; always produced from the current sources via `--regen`)

## Gotchas
- The Lua state does **not** open the `io` library; the report uses `print`
  (base lib → stdout). Don't reintroduce `io.*` here.
- `regen_expected.sh` skips commented-out lines, so disabled bindings are
  correctly excluded from the expected surface.
- To assert a new function's **behavior** (not just existence), add a
  `try(...)` line in `bin/data/bindcheck.lua`.
- Scope: TrussC core (`trussc::`) only — addon APIs (tcxBox2d/tcxOsc/…) are not
  part of the tcxLua surface.
- If you run this from a secondary worktree, make sure the build resolves
  `TRUSSC_DIR` to THAT worktree's core (the default preset points at this
  checkout) — otherwise you validate the wrong tcxLua.
